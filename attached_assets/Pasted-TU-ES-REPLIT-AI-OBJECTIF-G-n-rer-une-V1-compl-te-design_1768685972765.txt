TU ES REPLIT AI. OBJECTIF : Générer une V1 complète (design + code) du site vitrine de notre studio de création de sites web “premium accessibles”.
Le site doit être MOBILE-FIRST (priorité absolue), ultra fluide, avec des animations propres (subtiles, pas gadget), et une expérience UX/UI moderne.

========================
1) CONTEXTE & OBJECTIFS
========================
Nous lançons un service de création de sites web à prix cassés, avec abonnement obligatoire pour l’hébergement + maintenance.
Positionnement : studio créatif premium, ton transparent + légèrement décalé mais pro.
Cible : indépendants / TPE / PME (Belgique, France, Luxembourg), langue principale FR.
Promesse : “un site propre, rapide, au goût du jour, sans se ruiner”.

But principal : générer des leads qualifiés via un formulaire multi-étapes (et CTA clairs).
But secondaire : crédibilité (réalisations), transparence (packs + abonnement + règles), rassurance (process + FAQ).

Inspirations graphiques/animations (ne pas copier, s’en inspirer pour l’esprit) :
- minimalisme + animations clean
- typographie forte, sections très structurées, menu overlay, micro-interactions, marqueurs 01/02/03
- scrollytelling léger, reveal au scroll, sliders/projects

========================
2) STACK TECHNIQUE
========================
Choisir un stack robuste et rapide sur Replit :
- Next.js (App Router) + TypeScript
- TailwindCSS
- Animations : Framer Motion (reveal, transitions, menu overlay). Optionnel : simple “smooth scroll” léger.
- Pas de dépendances lourdes inutiles.
- SEO de base : metadata par page, OpenGraph.
- Accessibilité : aria, focus states, contraste OK, “prefers-reduced-motion” respecté.

Architecture :
- Site principalement en 1 page (landing longue) + pages légères : /realisations, /mentions, /privacy (ou sections ancrées si tu préfères).
- Multilingue : prévoir la structure technique pour 2 langues max (FR + EN) MAIS afficher par défaut FR. (Un toggle FR/EN en haut.) Contenus EN peuvent être simples placeholders.

========================
4) DESIGN SYSTEM (MOBILE FIRST)
========================
Style : premium minimal, beaucoup d’air, contrastes nets, typographie marquée.
- Couleurs : noir/blanc + 1 accent (ex: lime / electric blue / warm orange). Définir dans tailwind config.
- Typo : une sans moderne (ex Inter) + une serif optionnelle pour les titres (ou uniquement sans si plus simple). 
- Grille : sections claires, spacing généreux, max-width.
- Composants : boutons (primary/secondary/ghost), pills, cards, accordéon FAQ, badge “À partir de”, stepper pour formulaire.
- Micro-interactions : hover, press, reveal.

Mobile-first :
- Chaque section doit être parfaite sur mobile (375px) avant desktop.
- Menu : hamburger -> overlay plein écran très clean (avec animations) + CTA visible.
- CTA sticky sur mobile (petit bandeau “Demander mon site” qui apparaît après scroll, discret).

Animation rules (important performance) :
- Animations subtiles, pas de gros effets sur mobile.
- Désactiver ou réduire si prefers-reduced-motion.
- Révélations au scroll : fade + translate léger.
- Un “marquee” (texte qui défile) très léger (CSS) possible.
- Parallax : uniquement si léger et performant, sinon simuler par transform minimal.

6) FORMULAIRE MULTI-ÉTAPES (LEAD QUALIFICATION) — CRITIQUE
========================
Créer un composant <LeadWizard /> avec étapes, barre de progression, validation, UX nickel sur mobile.

Étapes proposées (avec logique):
Étape 1 — Ton activité :
- type de business (liste + “autre”)
- zone (BE/FR/LU, ville)
Étape 2 — Ton besoin :
- type de site : landing / vitrine / multi-pages
- nombre de pages (1 / 2-5 / 6+)
- besoin formulaire : oui/non + “simple” ou “multi-étapes”
Étape 3 — Langues :
- 1 ou 2 langues (max 2)
- langues (FR/EN/NL autre) mais bloquer à 2
Étape 4 — Domaine :
- j’ai déjà un domaine / non
- je veux une adresse mail pro : oui/non
Étape 5 — Timing & budget :
- délai : 24-48h / 1 semaine / flexible
- budget création : 199 / 299 / 499 / 749+ (si e-commerce)
Étape 6 — Coordonnées :
- nom, email, tel
- message libre

Sortie du wizard :
- afficher un écran “Résumé” avec :
  - pack recommandé (auto) :
    - si 1 page sans form -> 199
    - si 1 page avec form -> 299
    - si 2-5 pages -> 499
    - si e-commerce -> 749+
  - abonnement recommandé :
    - domaine à gérer -> 24,99
    - sinon -> 19,99
  - rappel : 2 tickets/mois, fair-use 5€
- CTA : “Envoyer ma demande” + “Réserver un appel”

Envoi :
- Créer une route API /api/lead
- Pour V1 : stocker les leads dans un fichier JSON local ou une DB simple (optionnel).
- BONUS : prévoir l’envoi par email via SMTP (nodemailer) avec variables d’environnement (ne pas hardcoder).
- Afficher message de succès + fallback si échec.
