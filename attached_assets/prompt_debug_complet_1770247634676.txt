Le problème mobile persiste après les modifications. Voici ce qui se passe maintenant :

NOUVEAUX SYMPTÔMES OBSERVÉS :
- Safari : Page reste blanche même après désactivation des animations FadeIn
- Chrome mobile : Page grise, une partie du contenu apparaît après 5 secondes, le reste ne charge jamais
- Le problème se produit aussi en navigation privée (donc pas un problème de cache)
- Sans interaction (pas de scroll), le contenu ne se charge pas

ANALYSE : Il y a PLUSIEURS problèmes qui se cumulent, pas juste FadeIn.

ACTIONS DE DÉBOGAGE IMMÉDIAT :

1. **AJOUTER UN DIAGNOSTIC COMPLET EN HAUT DU HTML** :
   Dans le <head> de index.html ou _document.js, ajoute ceci AVANT tout autre script :
   
   ```html
   <script>
     // Diagnostic mobile ultra-prioritaire
     window.mobileDebugger = {
       startTime: Date.now(),
       userAgent: navigator.userAgent,
       viewport: window.innerWidth + 'x' + window.innerHeight,
       errors: [],
       warnings: [],
       loadedScripts: [],
       failedScripts: []
     };
     
     // Capturer TOUTES les erreurs
     window.addEventListener('error', function(e) {
       window.mobileDebugger.errors.push({
         type: 'error',
         message: e.message,
         source: e.filename,
         line: e.lineno,
         col: e.colno,
         time: Date.now() - window.mobileDebugger.startTime
       });
       console.error('ERROR CAUGHT:', e);
     }, true);
     
     // Capturer les promesses rejetées
     window.addEventListener('unhandledrejection', function(e) {
       window.mobileDebugger.errors.push({
         type: 'promise_rejection',
         reason: e.reason?.toString(),
         time: Date.now() - window.mobileDebugger.startTime
       });
       console.error('PROMISE REJECTION:', e.reason);
     });
     
     // Forcer l'affichage après 2 secondes si rien n'apparaît
     setTimeout(function() {
       const bodyText = document.body.innerText.trim();
       const bodyVisible = document.body.offsetHeight > 100;
       
       if (!bodyVisible || bodyText.length < 50) {
         console.error('EMERGENCY: Content not visible!');
         
         // Afficher le diagnostic
         document.body.innerHTML = '<div style="padding:20px;font-family:monospace;background:#fff;color:#000;"><h1>🚨 DIAGNOSTIC MOBILE</h1><h2>Temps écoulé: ' + (Date.now() - window.mobileDebugger.startTime) + 'ms</h2><h3>Erreurs détectées:</h3><pre>' + JSON.stringify(window.mobileDebugger, null, 2) + '</pre><h3>Body height: ' + document.body.offsetHeight + 'px</h3><h3>Body text length: ' + bodyText.length + '</h3><button onclick="location.reload()" style="padding:10px;margin-top:20px;">Recharger</button></div>';
       }
     }, 2000);
     
     console.log('Mobile debugger initialized');
   </script>
   ```

2. **VÉRIFIER ET CORRIGER LE CSS** :
   Cherche dans ton CSS global si tu as :
   - `body { opacity: 0 }` ou `html { opacity: 0 }`
   - Des classes avec `display: none` qui s'appliquent sur mobile
   - Des `min-height: 100vh` qui peuvent causer des problèmes
   
   Ajoute ce CSS de sécurité au début de ton CSS principal :
   ```css
   /* URGENCE - Forcer la visibilité sur mobile */
   @media (max-width: 768px) {
     html, body {
       opacity: 1 !important;
       visibility: visible !important;
       display: block !important;
     }
     
     /* Forcer tous les éléments principaux à être visibles */
     main, .hero, .hero-content, section, article {
       opacity: 1 !important;
       visibility: visible !important;
       display: block !important;
     }
     
     /* Désactiver TOUTES les animations sur mobile temporairement */
     * {
       animation-duration: 0s !important;
       transition-duration: 0s !important;
     }
   }
   ```

3. **VÉRIFIER LE JAVASCRIPT** :
   Cherche dans ton code si tu as :
   - Des imports de modules qui peuvent échouer sur mobile
   - Du code qui attend un événement avant d'afficher le contenu
   - Des conditions qui cachent le contenu (`if (window.innerWidth > X)`)
   
   Commente temporairement TOUT le JavaScript non-essentiel :
   ```javascript
   // Désactive temporairement :
   // - Toutes les animations Framer Motion
   // - Tous les useEffect qui manipulent l'affichage
   // - Tous les lazy loading
   // - Tous les IntersectionObserver
   ```
